## Задание 1: Анализ приложения

⏱️ **20 минут**

**Шаг 1:** Скачать APK по ссылке: `[ссылка]`

**Шаг 2:** Установить приложение

- Если есть Android телефон — установи APK напрямую
- Если нет Android — открой на ноутбуке сайт https://www.browserstack.com/app-live или https://www.lambdatest.com/real-device-cloud/app-testing и запусти приложение в эмуляторе (регистрация бесплатная)

**Шаг 3:** Протестируй приложение — открой все экраны, попробуй все кнопки, посмотри как работает

**Шаг 4:** Запиши скринкаст видео 2-3 минуты где покажешь:

- Что делает приложение (главные функции)
- Какие баги или странности нашёл
- Твоё общее впечатление

⚠️ **ОБЯЗАТЕЛЬНО посмотри пример перед записью:** `[добавлю ссылку]` — делай похожий формат!

**Как записать скринкаст:**

- На телефоне: встроенная запись экрана (шторка уведомлений → "Запись экрана")
- На ноутбуке в эмуляторе: OBS Studio, QuickTime (Mac) или встроенная запись Windows (Win+G)

**Куда залить:** YouTube (unlisted) или Google Drive / Яндекс.Диск с открытым доступом **Формат:** Видео на русском языке **Куда отправить:** Ссылку на видео в финальную форму

## Задание 2: GitHub Pull Request

⏱️ **5 минут**

**Шаг 1:** Открой ссылку на репозиторий: `[добавлю ссылку на GitHub репо]`

**Шаг 2:** Если нет аккаунта GitHub — зарегистрируйся на https://github.com/signup (займёт 2 минуты)

**Шаг 3:** Найди в репозитории файл `info.md` (он будет в списке файлов на главной странице) и открой его

**Шаг 4:** Нажми кнопку с иконкой карандаша (✏️) справа вверху над текстом файла — она называется "Edit this file"

**Шаг 5:** В открывшемся редакторе заполни:

```
Меня зовут: [твоё имя]
Контакт: [твой Telegram или Email]
```

**Шаг 6:** Прокрути вниз страницы, там будет зелёная кнопка **"Propose changes"** (или "Commit changes") — нажми её

**Шаг 7:** На следующей странице нажми зелёную кнопку **"Create pull request"**

**Шаг 8:** Ещё раз нажми **"Create pull request"** для подтверждения

**Шаг 9:** Скопируй ссылку из адресной строки браузера (она будет выглядеть как github.com/название/pull/123) — это ссылка на твой Pull Request

**Куда отправить:** Эту ссылку в финальную форму

## Задание 3: Изучение ТЗ и тест по сервисам

⏱️ **1 час**

**Шаг 1:** Скачай техническое задание: `[добавлю ссылку]`

**Шаг 2:** Внимательно прочитай весь документ — там описаны разные сервисы (Amplitude, GitHub, Mixpanel и другие)

**Шаг 3:** Пройди тест по ссылке: `[добавлю ссылку на Google форму]`

- В тесте 30 вопросов про эти сервисы
- Вопросы конкретные, нужно знать детали — гугли, заходи на сайты сервисов, изучай
- Например: "Сколько стоит тариф Pro в Amplitude?", "Какая кнопка слева в главном меню GitHub?"

**Шаг 4:** Создай документ (Google Doc или Word) и напиши описание сервисов своими словами (1-2 страницы):

- Что делает каждый сервис из ТЗ
- Основные функции каждого (по 5-7 пунктов)
- Объясни простыми словами, как будто рассказываешь другу

**Как создать Google Doc:** Открой https://docs.google.com → нажми "+ Пустой документ" → пиши → потом "Настройки доступа" справа вверху → "Всем у кого есть ссылка" → скопируй ссылку

**Куда отправить:** Ссылку на Google Doc (или залей PDF на Google Drive с открытым доступом) в финальную форму
